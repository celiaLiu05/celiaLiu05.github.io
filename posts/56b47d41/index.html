开发流程

发表于2022-11-04|开发流程
|阅读量:

feature-PRODUCT-1902：基于master分支拉的。

开发结束需求提测，将开发分支feature-xxxxx-xx代码合入到默认测试环境分支test-a

测试结束上预发环境，首先需新建一个上线分支如release-a-{YYYY-MM-DD}，（日期为需求上线的实际日期），将开发分支feature-xxxxx-xx 代码合入到预发分支并通知测试

dev-a：开发环境

test-a：测试环境

release-2023-05-26：发布分支。发布完成后，由相应release 分支合并到 master分支

**hotfix-**：热修分支。修复代码

以上五个全部基于master分支拉

master

  • 创建并拉取分支：点加号，new branch，feature-PRODUCT-1981，create branch。项目里面切换到1981分支，git fetch
  • 开发完成之后提交代码：先拉取一下代码，点击左下角刷新，git fetch，git pull。往远程去推，注释是T-PRODUCT-1981 需求，commit&push
  • 合并分支的操作在gitlab上进行，加号，new merge request，左边的分支是开发分支，右边的分支是未来要合并的分支。合并如果有冲突，将test-a拉取到本地，将本地的分支往test-a上和，解决冲突，再将test-a推到远程。（不要在gitlab上解决冲突）（合并分支的操作在gitlab上进行）（新分支创建统一使用Gitlab工具）
    • git checkout test-a
    • git merge origin/feature-PRODUCT-1981
  • 合并分支完成，在jira上点击开发完成，今天，自测通过。提测
  • 天枢运维平台，新建上线单，链接发给测试人员
  • 测试没有问题，上预发，创建一个release-05-29分支，把feature分支合并到release分支上，合预发，发一个链接
