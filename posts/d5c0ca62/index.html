理解js中的对象

发表于2022-11-04|js
|阅读量:

创建方式

  1. let obj = new Object()，再向obj身上添加属性
  2. 对象字面量，const obj = {name: 'ljw', age: 20}

属性的类型

数据属性是直接存储属性的值，而访问器属性使用 getter 和 setter 方法进行属性值的读取和写入。

数据属性

  • configurable，是否可以被删除，默认值为true

当希望防止某个属性被删除或修改特性时，可以将configurable设置为false。这可以用于保护对象的关键属性，防止它们被意外删除。

  • enumberable，是否可以枚举，默认值为true

当希望在遍历对象属性时控制某个属性是否可见时，可以使用enumerable属性。通过将enumerable设置为false，可以防止属性被for...in循环或Object.keys()等方法枚举到。

  • writable，是否可以被修改，默认值为true
  • value，属性实际的值，默认值为undefined

以上都是使用Object.defineProperty()定义属性，也可以在对象字面量中使用属性描述符

访问器属性

  • configurable，是否可以删除并重新定义

  • enumberable，是否可以通过for in 循环

  • Get，读取属性时调用的函数

  • Set，写入属性时调用的函数

定义属性

  • Object.defineProperty()

它的作用包括：

  1. 定义新属性：使用 Object.defineProperty() 方法可以向对象添加新属性，并指定该属性的特性。可以定义数据属性或访问器属性。
  2. 修改属性特性：如果对象已经有属性，可以使用 Object.defineProperty() 方法修改该属性的特性，例如可写性、可枚举性和可配置性。

Object.defineProperty(obj, prop, descriptor)

obj：要定义或修改属性的目标对象。

prop：要定义或修改的属性名称。

descriptor：属性描述符对象，包含要定义或修改的属性的特性和行为。

  • Object.defineProperties()

Object.defineProperties(obj, descriptors)

obj：要定义或修改属性的目标对象。

descriptors：一个包含属性描述符对象的对象。每个属性描述符对象定义一个要定义或修改的属性及其特性。

读取属性的特性

  • Object.getOwnPropertyDescriptor()，获取指定对象上 指定属性的 属性描述符

Object.getOwnPropertyDescriptor(obj, prop)

obj：要获取属性描述符的目标对象。

prop：要获取属性描述符的属性名称。

返回值是一个属性描述符对象，其中包含了指定属性的特性。

  • Object.getOwnPropertyDescriptors()，获取指定对象上 所有属性 的完整属性描述符

Object.getOwnPropertyDescriptors(obj)

obj：要获取属性描述符的目标对象。

返回值是一个包含指定对象上所有属性的完整属性描述符的对象。

合并对象Object.assign()

Object.assign()，用于将所有可枚举属性的值从一个或多个源对象分配到目标对象。它将返回目标对象。

Object.assign(target, ...sources)

  • target：目标对象，将源对象的属性复制到该对象中。
  • sources：一个或多个源对象，从这些对象中复制属性到目标对象中。

创建新对象Object.create()

Object.create()， 以一个现有对象作为原型，创建一个新对象

Object.create(proto, propertiesObject)

  • proto 新创建对象的原型对象。
  • propertiesObject 可选，如果该参数被指定且不为 undefined，则该传入对象可枚举的自有属性将为新创建的对象添加具有对应属性名称的属性描述符。

对象相等判定 Object.is()

Object.is()，确定两个值是否为相同值

Object.is(value1, value2)

如果以下其中一项成立，则两个值相同：

  • 都是 undefined
  • 都是 null
  • 都是 true 或者都是 false
  • 都是长度相同、字符相同、顺序相同的字符串
  • 都是相同的对象（意味着两个值都引用了内存中的同一对象）
  • 都是 BigInt 且具有相同的数值
  • 都是 symbol 且引用相同的 symbol 值
  • 都是数字且
    • 都是 +0
    • 都是 -0
    • 都是 NaN
    • 都有相同的值，非零且都不是 NaN

Object.is() 与 == 的区别。

== 运算符在测试相等性之前，会对两个操作数进行类型转换（如果它们不是相同的类型），这可能会导致一些非预期的行为，例如 "" == false 的结果是 true，但是 Object.is() 不会对其操作数进行类型转换。

Object.is() 也不等价于 === 运算符。

Object.is() 和 === 之间的唯一区别在于它们处理带符号的 0 和 NaN 值的时候。=== 运算符（和 == 运算符）将数值 -0 和 +0 视为相等，但是会将 NaN 视为彼此不相等。

Object.prototype.isPrototypeOf()

isPrototypeOf(object)用于检查一个对象是否存在于另一个对象的原型链中

isPrototypeOf(object)

  • object，要搜索object的原型链。

调用 isPrototypeOf() 方法的对象（即 this）是否位于 object 的原型链中

Object.getPrototypeOf()

Object.getPrototypeOf() 返回指定对象的原型。

Object.getPrototypeOf(obj)

  • obj，要返回obj的原型。

Object.prototype.hasOwnProperty()

Object.prototype.hasOwnProperty()返回一个布尔值，表示对象自有属性（而不是继承来的属性）中是否具有指定的属性。

hasOwnProperty(prop)

  • prop，要测试的属性的字符串名称或者 Symbol。
